# 格式修正 Skill

检查用户提供的代码片段（通常是部署脚本、配置文件、heredoc 块等），识别并修正所有格式错误，输出可直接复制使用的修正版本。

## 输入

$ARGUMENTS

如果没有提供参数，检查当前 IDE 选中的代码。

## 检查规则

按以下优先级逐项检查：

### 1. Heredoc 格式
- heredoc 内容（`<< 'EOF'` / `<< 'COMPEOF'` 等）必须**顶格写**，不能有多余前导空格
- heredoc 终止符（`EOF`、`COMPEOF`、`PYEOF` 等）必须**顶格写**（除非使用 `<<-` + Tab）
- heredoc 之后的独立命令（如 `chmod`）不能被缩进吞掉

### 2. YAML / Docker Compose 缩进
- `services:` 必须在顶层（0 缩进）
- 层级关系：service名 2格 → 属性 4格 → 子属性 6格 → 子子属性 8格
- 同级 key 缩进必须一致（如 `environment:` 下所有变量同级）
- `command:` 长值换行时使用 `>-` 折叠语法，不能裸断行

### 3. Python 缩进
- 顶层代码必须从第 0 列开始
- 三引号字符串内的内容缩进要与目标文件格式匹配，而非与 Python 代码对齐

### 4. Shell 脚本
- `#!/bin/bash` shebang 必须在文件第 1 列
- `.env` 文件的 key 前不能有空格（dotenv 解析器敏感）

### 5. JavaScript / TypeScript
- 文件级代码从第 0 列开始
- 内部缩进统一 2 空格

## 输出格式

1. **问题清单**：用编号列出发现的每个问题，说明位置和原因（简洁，每条 1 行）
2. **修正后完整代码**：放在代码块中，可直接复制粘贴使用
3. 如果没有格式问题，回复「格式正确，无需修改」
