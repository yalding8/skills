## §表单 API 坑（建单前必读，全部实测踩过）

1. **phone 类型字段进不了新版表单** → 要进表单的电话一律 text。
2. `+form-questions-create` 会**新建字段**而非绑定同名已有字段 → 先建字段，表单题目用
   PATCH 已有 field 开 visible/required，不用 create。
3. 同批建题会**倒序插入** → 题目重排走原生 `PATCH /open-apis/bitable/v1/apps/:app/tables/:tbl/forms/:form/fields/:fld`
   传 `pre_field_id`（同接口可改 visible/required/title/description）。
4. **公开填写**：`PATCH .../forms/:form_id` `{"shared":true}`，开启后 `shared_limit` 自动
   `anyone_editable`（合法值就这个，`anyone_can_submit` 报 99992402）；`+form-update` shortcut
   不支持分享设置，必须走 `lark-cli api` 原生 PATCH。`submit_limit_once:false`=可重复提交。
5. 旧 UI 建的表单不支持 /forms questions API（沈阳同事表教训）；回收中途**追加题目安全**
   （即时生效不用重发链接），但老提交无新字段——播报模板对追加字段 `{% if %}` 整行隐藏。
6. 表单题即表字段：改字段类型（text→select 白名单内）表单题自动跟随。
7. 测试行处理默认「必填题放行闸」（播报 trigger 用必填列 nonempty，手工测试行不填即不触发），
   别物理删别人数据。
